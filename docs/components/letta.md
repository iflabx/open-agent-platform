# Letta

> 类型：长期记忆 Agent 能力
>
> 复用规则：条件引入的组件说明页
>
> 所属层：Agent 编排层

## 当前定位

`Letta` 用于长期记忆、持续状态和个体化上下文管理，不作为所有应用的默认运行时。

## 主要职责

- 长期记忆管理
- 持续上下文维护
- 个体化助手能力

## 与其他组件关系

- 作为编排层的条件补强能力，用于补足长期记忆和持续状态管理。
- 可与 [LangGraph](/components/langgraph)、[LangChain](/components/langchain) 等运行时配合使用。
- 上游通常由门户层或编排层调用，不直接作为统一入口。
- 记忆数据、权限语义和删除策略需要与治理层和数据层联动。

## 适合场景

- 长周期协作助手
- 需要持续上下文的专业助手
- 高价值用户陪伴式 Agent

## 边界

- 不用于所有标准场景
- 不替代主平台中枢和知识平面

## 采用规则

- `Letta` 只在确有长期记忆、持续状态和个体化上下文要求时引入。
- 标准问答、短会话和一次性流程不默认启用 `Letta`。
- 引入后要明确哪些内容属于长期记忆，哪些仍应留在知识层或业务系统中。

## 治理注意点

- 长期记忆要有清晰的保留、更新、删除和审计规则。
- 个体化状态不能绕过统一权限和数据分类分级要求。
- 需要重点关注记忆污染、陈旧上下文和越权记忆复用问题。
- 发布前应验证记忆写入、读取、清理和人工纠偏流程。
