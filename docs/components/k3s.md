# K3s

> 类型：容器运行与服务编排平台
>
> 复用规则：稳定复用的组件说明页
>
> 所属层：基础设施层

## 当前定位

`K3s` 是当前方案的平台运行底座。

## 主要职责

- 平台服务部署
- 服务编排与扩容
- 多环境管理
- 配置与密钥挂载

## 与其他组件关系

- 为门户层、编排层、数据治理层、知识层、模型层和观测层提供统一运行底座。
- 与 [PostgreSQL](/components/postgresql)、[Redis](/components/redis)、[MinIO](/components/minio) 等状态型服务共同构成平台基础设施。
- 与企业现有 `KMS / 证书体系`、网络和存储体系配套使用。
- 为发布、扩容、回滚和环境隔离提供基础能力。

## 适合场景

- 私有化智能体平台部署
- 中小到中大型集群规模的企业内部运行
- 需要快速收敛 Kubernetes 运维复杂度的场景

## 边界

- 不承载业务流程、权限策略和应用逻辑
- 不替代数据库、缓存和对象存储
- 不替代上层发布治理与观测策略

## 采用规则

- 当前方案默认以 `K3s` 作为容器运行与服务编排底座。
- 当规模、合规或平台治理复杂度显著上升时，再单独评估向标准 Kubernetes 体系演进。
- 基础设施层应保持收敛，不在早期同时维护多套正式容器底座。

## 治理注意点

- 开发、测试、生产环境需要明确隔离。
- 集群升级、备份、恢复和扩容要形成标准化操作流程。
- 密钥、证书和配置分发不能散落在业务镜像或应用代码中。
- 状态型服务和无状态服务的资源规划应区别设计。
