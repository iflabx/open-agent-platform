## 从蓝图到生产

<p class="docs-section-intro">参考现代开发文档首页的组织方式，这里按“理解 -> 设计 -> 验证 -> 实施”给出最短阅读链路。</p>

<div class="docs-journey-grid">
  <a class="docs-step-card" href="/guide/">
    <span>1</span>
    <strong>理解问题</strong>
    <p>先看方案综述，明确建设目标、设计原则和优先落地场景。</p>
  </a>
  <a class="docs-step-card" href="/architecture">
    <span>2</span>
    <strong>建立架构</strong>
    <p>看总体架构、数据治理层和安全治理，确定控制面、运行面和数据面边界。</p>
  </a>
  <a class="docs-step-card" href="/protocol-standards">
    <span>3</span>
    <strong>统一标准</strong>
    <p>梳理 <code>MCP</code>、<code>A2A</code>、身份接入、接口标准、授权规范和可观测语义。</p>
  </a>
  <a class="docs-step-card" href="/deployment">
    <span>4</span>
    <strong>进入实施</strong>
    <p>对照组件清单、许可证说明、部署建议和路线图，推进到试点和生产。</p>
  </a>
</div>
