# 典型场景

## 场景一：内部知识助手

### 目标

让员工通过自然语言快速查询制度、流程、产品说明、售后知识和项目资料。

### 推荐能力

- 文档同步与增量索引
- 基于部门、角色和项目的权限过滤
- 多轮对话上下文管理
- 引用来源展示与回答置信度提示

### 落地重点

这个场景通常最适合作为企业第一阶段试点，因为价值明确、数据边界相对清晰、与现有流程冲突较少。

## 场景二：服务台智能分诊

### 目标

对内部 IT、HR、财务或客户服务工单进行自动分类、问答和分派，降低一线支持重复工作量。

### 推荐能力

- 意图识别与表单补全
- 工单系统双向集成
- 规则引擎与模型协同判定
- 高风险操作的人机确认

### 落地重点

这个场景需要把“回答”升级为“动作”。因此除了模型表现，审计、重试、超时、权限和回滚逻辑会变得很关键。

## 场景三：流程自动化 Agent

### 目标

将跨系统流程拆成若干可观测、可中断、可恢复的任务链，例如销售线索处理、采购审批前校验、知识库发布流程等。

### 推荐能力

- 状态机驱动的任务编排
- 事件触发和补偿机制
- 节点级日志和任务回放
- 人工审批插槽

### 落地重点

如果流程链路较长，建议把工作流层与大模型调用层解耦，避免每次流程变化都牵动底层智能能力。

## 场景四：数据分析助手

### 目标

让业务人员以自然语言生成查询、解释指标异常、辅助汇总分析结论。

### 推荐能力

- SQL 生成与安全校验
- 元数据字典和指标口径管理
- 结果解释、图表摘要和追问
- 数据权限与沙箱隔离

### 落地重点

这一类场景的风险不在“不会答”，而在“答得看起来合理但其实错误”。上线前必须建立稳定的评测集和人工验真流程。
